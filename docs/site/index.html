🔍

Keyword Search

Port 8080 / gRPC 50051

Trigram-indexed, AST-aware search. Zero cold-start. Best for exact code patterns, function calls, and variable names.

  • Roaring bitmap trigram index
  • Tree-sitter symbol extraction (12+ languages)
  • PageRank-style import graph scoring
  • Regex with trigram acceleration
  • Parallel search via rayon
  • File watcher & incremental indexing
  • Index persistence (disk save/load)
  • Symbols-only search mode

fn main · class.*Handler · import useState

🧠

Semantic Search

Port 8081 / gRPC 50052 · optional

Understands code meaning. Best for natural language queries that don't map to exact token matches.

  • TF-IDF (fast, zero dependencies)
  • CodeBERT / UniXcoder ML embeddings
  • HNSW vector index for similarity search
  • Code-aware chunking (functions, classes)
  • ONNX Runtime inference
  • Model SHA-256 integrity checks

"retry with exponential backoff" · "validate user input"

Why fast_code_search?

Unlike CLI tools or disk-based indexes, it runs as an always-on, in-memory server

🚀

Sub-millisecond

Index always hot in RAM — no disk I/O on queries. Memory access is 100–1000× faster than SSD.

📦

10 GB+ Codebases

Memory-mapped files via memmap2. Handles 600k+ files without loading everything into RAM.

⚙️

Symbol Awareness

Tree-sitter parses Rust, Python, JS/TS, Go, C/C++, Java, C#, Ruby, PHP, Bash and more.

📡

Dual API

gRPC streaming (tonic) for IDE plugins. REST/JSON (axum) for scripts and web UIs.

📈

Smart Scoring

Symbol defs (3×), exact match (2×), src/lib dirs (1.5×), PageRank import boost.

🔄

Live Indexing

File watcher tracks filesystem changes. Incremental re-indexing keeps the index fresh.

💾

Index Persistence

Save the full index (v3 format with symbols & deps) to disk. Checkpoints during large builds.

🌐

Embedded Web UI

Browser-based search on port 8080. Static assets compiled into the binary — zero deployment friction.

Getting Started

1

Install from source

git clone https://github.com/jburrow/fast_code_search.git
cd fast_code_search
cargo build --release
2

Configure

# Generate a default config file
./target/release/fast_code_search_server --init config.toml

# Edit to point at your codebase
# [indexer]
# paths = ["/path/to/your/repo"]
3

Run

./target/release/fast_code_search_server --config config.toml

# Web UI available at http://localhost:8080
# gRPC available at localhost:50051
4

Search

# REST API
curl "http://localhost:8080/api/search?q=fn+main&max=10"

# Symbols only
curl "http://localhost:8080/api/search?q=SearchEngine&symbols=true"

# Regex
curl "http://localhost:8080/api/search?q=async+fn.*Handler®ex=true"
